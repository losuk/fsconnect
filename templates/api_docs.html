FSConnect API Documentation

Welcome to the FSConnect API documentation. Here you'll find all the information needed to integrate and use our API effectively.

API Overview

The FSConnect API allows developers to integrate advanced AI capabilities into their applications. Below are some of the key endpoints available:

1. Summarize Content

Endpoint: POST /api/summarize

Request Body:

{
    "text": "Your text to be summarized"
}

Response:

{
    "summary": "The summarized content here."
}

Quickstart Guide

To get started with the FSConnect API, follow these steps:

  1. Install the FSConnect API package using one of the following commands:
    • For Python: 
      pip install fsconnect
    • For JavaScript: 
      npm install fsconnect
  2. Set up your API key:
    1. Visit the API Keys page on our website.
    2. Log in to your account.
    3. Click on "Generate API Key" and copy the generated key.
    4. Store your API key securely.
  3. Make your first API call:

    4. Here’s an example of how to use the FSConnect API in both Python and JavaScript:

    • For Python: 
      
import fsconnect

# Set your API key
api = fsconnect.FSConnect(api_key='YOUR_API_KEY')

# Make a request
response = api.summarize("Your text to be summarized")
print(response)
    • For JavaScript: 
      
const fsconnect = require('fsconnect');

// Set your API key
const api = new fsconnect.FSConnect('YOUR_API_KEY');

// Make a request
api.summarize("Your text to be summarized").then(response => {
    console.log(response);
});

API Keys

Your API key is a unique identifier used to authenticate your requests to the FSConnect API. Follow the instructions below to create and manage your API keys.

Generating an API Key

To generate a new API key, follow these steps:

  1. Log in to your FSConnect account.
  2. Navigate to the API Keys section in your account settings.
  3. Click on the Generate New API Key button.
  4. Your new API key will be displayed. Make sure to copy it and store it in a secure location, as you will not be able to view it again.

Note: Each user is allowed to generate up to 5 API keys at a time.

Managing Your API Keys

You can manage your existing API keys in the same section where you generated them. Here’s how:

  • View API Keys: All your active API keys will be listed along with their creation date and status.
  • Regenerate API Keys: If you think your API key has been compromised, you can regenerate it by clicking on the Regenerate button next to the respective key.
  • Delete API Keys: To remove an API key, click on the Delete button. Be careful as this action is irreversible.

Security Practices for API Keys

To keep your API keys secure, follow these best practices:

  • Never share your API keys publicly, including in code repositories.
  • Store your API keys securely, such as in environment variables or encrypted storage.
  • Regularly review and rotate your API keys to minimize the risk of exposure.
  • Monitor your API usage and revoke any keys that are no longer needed or appear suspicious.

Troubleshooting

If you encounter any issues with your API keys, check the following:

  • Ensure you are using the correct API key in your requests.
  • Verify that your key has not been deleted or regenerated.
  • Contact FSConnect support if you continue to experience problems.

Available Models

The FSConnect API supports various models for different tasks. Below are the current models available:

1. Summarization Model

This model is optimized for summarizing long texts. It uses advanced algorithms to identify key points and condense information effectively.

Usage Example:

POST /api/summarize

2. Translation Model

This model can translate text from one language to another. It supports multiple languages and dialects, ensuring accurate and context-aware translations.

Supported Languages:

  • English
  • Spanish
  • French
  • German
  • Italian
  • Chinese
  • Japanese
  • And more...

Usage Example:

POST /api/translate

3. Sentiment Analysis Model (Upcoming)

This upcoming model will allow you to analyze the sentiment of a given piece of text, categorizing it as positive, negative, or neutral.

Planned Features:

  • Real-time sentiment scoring.
  • Support for multiple languages.
  • Integration with existing endpoints for contextual analysis.

4. Text Generation Model (Upcoming)

This upcoming model will generate human-like text based on a given prompt. It will be useful for various applications, including chatbots and content creation.

Planned Features:

  • Customizable prompts for tailored content generation.
  • Ability to maintain context over long conversations.
  • Support for multiple writing styles and tones.
